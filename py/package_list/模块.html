模块和包

模块和包(包列表入口)

在python中，模块是个核心概念

每一个python程序都是由内建函数模块和外部模块组成的，就好比一个个不同的工具包，里面包含了不同功能的工具

每次要使用工具包(模块)时，必须先找到工具包(模块名)，再取其中你所需要的工具(函数)进行使用

每一个py模块和pyc文件都可以被当成一个模块来用，在模块中定义的 函数，全局变量 都可以被当作模块的参数进行调用

但如果想要一个文件以模块的形式被调用，那么其文件名就必须满足标 识符命名规则 而且尽量不要和系统内置模块重名

模块导入操作

import 模块名

  • 读取顺序：

    ./ 当前目录

    %path% 系统环境目录

别名

import 旧模块名 as 新模块名 # 符合大驼峰

  • 注意: 模块别名的使用情况

    有同名模块的时候, 后导入的模块会覆盖之前的模块, 这时候就可以对其进行重命名

    模块名太长, 太复杂时, 可以利用别名来简化模块名

局部导入

from 模块名 import 某个工具
  • 注意: 局部导入后, 可以不用别名.的方式来调用工具, 直接使用即可 (这也就产生了from module import *的方式来简化模块的使用)

__name__属性

__name__就是在兼顾文件开发和测试性的一个内置属性

打开一个python文件，输入

print(__name__)

就会返回

__main__

这是因为你在直接运行某个python文件的时候，解释器会把该文件当作主程序，并且赋予这个文件一个__main__属性

  • 原因

    在导入一个python文件的时候，解释器会把文件主程序完全运行一遍，这就导致如果你之前保留的测试代码，也会重新运行

但如果你不想运行测试代码，又想保留他们，就把主程序除定义以外的运行代码全部放到属性名称判断下面

if __name__ = '__main__':
    pass

导入文件时解释器会给导入文件的__name__属性定义为导入文件本身的名称, 这样最后结尾的if就为False，其后方的代码就不会被解释运行

包

包，就是很多个模块的组合一个，包不是一个文件，而是一个模块的目录

包的文件结构：以一个package包为例

  • /package

    __init__.py module_1.py module_2.py module_n.py modules_rquire.py

其中有个很特殊的文件__init__.py

其内容如下：

# 从 当前目录 导入 模块_n
from . import module_1
from . import module_2
from . import module_n

可以看作是包的初始化文件，其中的module可以看作是导入后直接能被执行的模块文件

modules_require可以看作是模块的小依赖

只有在__init__.py中有的模块才能用包进行导入

制作和发布包

先创建一个setup.py

内容：

from distutils.core import setup

setup(
    name="包名"
    version="版本号"
    description="简短描述"
    long_description="完整描述"
    author="作者"
    author_email="邮箱"
    url="主页url"
    py_modules=["包名.模块名","..."]
)

  • 构建模块命令

      python3 setup.py build

  • 生成发布压缩包

      python3 setup.py sdist

  • 安装外来包

      # 先解压
  unzip [file_name]
  tar -zxvf [file_name]
  # 安装
  python3 setup.py install

  • 想要卸载，直接删除对应包即可

    找不到位置的话可以尝试使用module.__path__的属性来查找
