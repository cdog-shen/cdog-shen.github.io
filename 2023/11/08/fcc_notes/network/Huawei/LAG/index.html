华为链路聚合以及端口模式配置

端口模式详解

端口的模式可用分成4种

access

access通常用于PC与SW的连接端口, 只会转发对应vlan的数据包

trunk

trunk模式通常是SW和SW之间, SW和RA之间网络设备的连接

在trunk模式下会转发所有/自己所处的vlan标签(也可以手动自己配置可用的vlan)

hybrid

可以看作是access和trunk的混合模式(比较通用)

dot1q-tunnel

Dot1q-tunnel端口，该端口进来的报文，无论当前是否携带VLAN tag，都会叠加端口的默认VLAN在外层。

QINQ能够实现私网VLAN透传到公网，达到二层透透传的应用效果，同时保证用户原有的VLAN功能与特性。用户可以规划自己私网VLAN ID，不会导致和公网VLAN ID冲突。它能解决日益紧缺的公网VLAN ID资源问题，为小型城域网或企业网提供一种简单的二层透传解决方案。

这个模式通常是路由的子接口所处模式, 用于vlan间互通

需要和trunk聚合一起使用

实现方法

  • 先开启SW的Vlan功能, 并配置对应接口

vlan 10 开启vlan10(左支vlan)

vlan 20 开启vlan20(右支vlan)

vlan 1 开启vlan1(聚合vlan)

  • 给对应端口配置vlan

interface <interfacetype> <interfacenum>

进入一个端口

port link-type access

port default vlan <a running vlan num>

更改ge0/0/2和ge0/0/3为access, 2对应vlan10, 3对应vlan20

port link-type trunk

更改ge0/0/1为trunk端口

port trunk pvid vlan 1

将其绑定到之前你指定的聚合vlan(不然其他vlan的会被过滤)

port trunk allow-pass vlan all

允许所有的vlan通过

  • 单臂路由配置(可以说是将一个接口虚拟化成多个逻辑接口)

int g 0/0/0.1

进入路由的单臂接口 0 的 1 号子接口

dot1q termination vid 20

绑定其vlan为20

ip address 192.168.1.254 255.255.255.0

指定子接口的ip地址(对应pc的网关)

arp broadcast enable

启用arp广播

同样根据上述指令配置一个属于vlan10的子接口2, IP地址为vlan10的网关

完成上述操作后, RA的路由表上就会出现两个子接口的路由信息, 这样一条物理链路也可以实现跨Vlan通信了

链路聚合的话就是:

interface eth-trunk <0-63>

undo portswitch

ip address X.X.X.X <mask>

创建一个聚合接口, 并且指定接口

interface <interfacetype> <interfacenum>

再进入物理接口

eth-trunk 1

绑定指定聚合链路就行了
